Diario per Applicazioni Web, a.a. 2009/10

Matteo Vaccari > Applicazioni Web

Tutte le letture e gli esercizi che non siano esplicitamente marcati come "facoltativi" sono parte del programma di esame.

I lucidi che potete trovare in questa pagina non sono un sostituto per il libro di testo o per gli altri materiali che vi segnalo. I lucidi non contengono un discorso completo, ma solo una traccia, che è comprensibile soltanto nel contesto di quello che dico a lezione quando li proietto.

Lo scopo di rendervi disponibili le slide è di lasciarvi una traccia dei discorsi che abbiamo fatto a lezione. Per chi non era presente, l'unica utilità delle slide è come promemoria degli argomenti che sono stati trattati.

Insomma, ripeto: non limitate il vostro studio ai lucidi!

  • 15-12-2009 Lezione 3
    Introduzione a Ruby
    • Lucidi
    • Risolvete gli esercizi che trovate nei lucidi, oppure completate il kata del supermercato, che abbiamo fatto in classe. Eravamo rimasti che la classe Prices deve implementare la responsabilità di restituire il costo di n istanze di un prodotto. Il test passa, ma i valori sono cablati nel codice, non vengono presi dagli input. Poi bisogna aggiornare la classe CheckOut per usare il nuovo metodo get_price di Prices invece dell'operatore []. (Per scaricare il progetto da github, installare git (per esempio su Ubuntu: "apt-get install git-core") e poi esegui
      • git clone git://github.com/xpmatteo/supermarket-kata-in-ruby.git
      • cd supermarket-kata-in-ruby
      • git checkout origin/in-class-2009-11-15
    • Studiare l'appendice A, "Introduction to Ruby" del libro di Rails.
    • Facoltativo: Un'ottima guida di riferimento è Programming Ruby, la cui prima edizione è consultabile liberamente online.
    • Facoltativo: imparara Ruby con i Ruby Koans; è un tutorial interattivo. (Per scaricare il progetto da github, installare git (per esempio su Ubuntu: "apt-get install git-core") e poi esegui "git clone git://github.com/edgecase/ruby_koans.git")

  • 20-01-2009 Lezione 8
    (Lezione doppia). Sessioni, cookies, autenticazione, protezione dell'accesso.
    • Lucidi
    • Studiare il capitolo 11 (Administration) del libro di Rails.
    • Ripassare Greenspun, capitolo 2, Basics.
    • Esercizio svolto in classe: http://github.com/xpmatteo/blog. Completare con le seguenti feature:
      • Un blog post deve poter avere commenti
      • Solo gli utenti registrati possono commentare
      • Sulla pagina personale degli utenti (es. /users/show/123) devono vedersi i link a tutti i post che hanno fatto, e a tutti i loro "amici"
      • Aggiungere il ruolo "admin". Un admin può listare tutti gli utenti del sito e vedere le loro email
      • Aggiungere la possibilità di chiedere l'"amicizia" di un altro utente. L'utente destinatario può approvare la richiesta oppure negarla.
      • Sulla pagina personale dell'utente loggato, devo poter vedere le mie richieste di amicizia
